  <action>
1. **Create `src/components/admin/image-manager/og-image.ts`:**
   - Export `generateOGImage(file: File | Blob): Promise<File>`
   - Use `createImageBitmap(file)` to decode the image
   - Create a canvas at 1200x630
   - Draw the image with cover-fit (crop to fill, centered):
     - Calculate scale = `Math.max(1200 / img.width, 630 / img.height)`
     - Center the scaled image on canvas
   - Export as JPEG with quality 0.75 (target under 200KB for WhatsApp)
   - Return as `new File([blob], 'og-cover.jpg', { type: 'image/jpeg' })`
   - If resulting blob > 200KB, retry with quality 0.6

2. **Add `uploadOGImage` server action to `src/actions/images.ts`:**
   - Signature: `uploadOGImage(propertyId: string, formData: FormData)`
   - Storage path: `{propertyId}/og-cover.jpg` (fixed name, upsert: true to overwrite previous)
   - Upload to `property-images` bucket with `upsert: true`
   - No DB row needed -- OG image is derived, not a user-managed photo
   - Return public URL for the OG image

3. **Integration point (for Plan 02's ImageManager):**
   - When `setCoverImage` is called successfully in ImageManager, also:
     a. Fetch the cover image file from its Supabase Storage URL
     b. Pass to `generateOGImage()` to create the 1200x630 variant
     c. Upload via `uploadOGImage` server action
   - This integration happens in ImageManager's cover action handler. If Plan 02 is implemented first, add a comment `// TODO: generate OG image on cover change` that this plan fills in. If this plan runs first, export a helper function `handleCoverWithOG(propertyId, imageUrl)` that Plan 02 can call.
   - Since both plans depend on 03-01 and may run in parallel, use a decoupled approach: export `generateAndUploadOG(propertyId: string, imageUrl: string)` from og-image.ts that fetches the image, generates OG, and calls uploadOGImage. ImageManager calls this function after setting cover.
