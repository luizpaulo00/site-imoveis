**ImageDropzone** (`image-dropzone.tsx`):
- "use client" component receiving `onFilesSelected: (files: File[]) => void` and `disabled?: boolean` props
- Styled drop area with dashed border, Upload icon (lucide), text "Arraste fotos aqui ou" + Button "Selecionar fotos"
- Hidden `input[type=file]` with `accept="image/jpeg,image/png,image/webp,image/heic,image/heif"` and `multiple`
- Handle native HTML5 dragover/drop events (NOT @dnd-kit -- this is file drop, not sortable)
- Visual feedback on dragover (border color change to primary)
- When disabled (during upload), show reduced opacity and prevent interaction
- PT-BR labels throughout

**ImageThumbnail** (`image-thumbnail.tsx`):
- "use client" component receiving `image: { id, storage_path, position, is_cover }`, upload status props, and action callbacks
- Aspect-ratio square thumbnail using `next/image` or `<img>` with Supabase Storage public URL
- **Cover indicator:** Star icon (lucide `Star`) in top-left corner. Filled/highlighted when `is_cover`. Click calls `onSetCover(imageId)`.
- **Delete button:** X icon (lucide `X`) in top-right corner. Click opens AlertDialog (same pattern as property delete) with "Excluir esta foto?" / "Esta acao nao pode ser desfeita" / Cancel + Confirm. Confirm calls `onDelete(imageId)`.
- **Cover highlight:** When `is_cover`, show ring-2 ring-primary border
- **Upload progress overlay:** When status is 'compressing' or 'uploading', overlay with Progress bar (shadcn). Show percentage during compression, indeterminate during upload. When 'error', show red overlay with error message.
- When status is 'done' or image is already uploaded, show normal thumbnail with hover overlay for actions

**ImageGrid** (`image-grid.tsx`):
- "use client" component wrapping thumbnails in @dnd-kit sortable context
- Import DndContext, closestCenter, SortableContext, rectSortingStrategy from @dnd-kit
- Grid layout: `grid grid-cols-2 md:grid-cols-3 gap-3`
- Each thumbnail wrapped in useSortable with image.id as sortable id
- CSS.Transform.toString for drag animation
- `onDragEnd`: compute new order using arrayMove, call `reorderImages` server action, update local state optimistically
- Mix uploaded images and uploading queue items in the grid (queue items are not sortable, appear at the end)

**ImageManager** (`image-manager.tsx`):
- "use client" component receiving `propertyId: string` and `initialImages: ImageRecord[]`
- Uses `useImageUpload(propertyId)` hook from Plan 01
- State: manages list of uploaded images (from initialImages), merges with upload queue
- Renders:
  1. Header: "Fotos" title + counter badge "3/15 fotos" (warn color when >= 13)
  2. ImageDropzone (disabled when isUploading or at limit)
  3. ImageGrid with all images (uploaded + uploading)
- **Cover action:** calls `setCoverImage` server action, updates local state optimistically
- **Delete action:** calls `deleteImage` server action, removes from local state. If deleted was cover, server auto-promotes next image.
- **Reorder action:** handled by ImageGrid's onDragEnd
- Wrap in shadcn Card with CardHeader + CardContent for visual consistency with other form sections
