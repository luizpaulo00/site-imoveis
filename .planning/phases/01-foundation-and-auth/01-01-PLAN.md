<task type="auto">
  <name>Task 2: Create Supabase clients, auth middleware, database schema, and initial tests</name>
  <files>src/lib/supabase/server.ts, src/lib/supabase/client.ts, src/middleware.ts, src/types/database.ts, supabase/schema.sql, src/__tests__/middleware.test.ts, src/__tests__/validations.test.ts</files>
  <action>
1. Create `src/lib/supabase/server.ts` -- Server-side Supabase client using the cookie pattern from @supabase/ssr. Use `createServerClient` with `getAll`/`setAll` cookie methods (NOT individual get/set/remove -- those are deprecated). Import `cookies` from `next/headers`. The function MUST be async because `cookies()` returns a Promise in Next.js 15.

2. Create `src/lib/supabase/client.ts` -- Browser-side Supabase client using `createBrowserClient` from @supabase/ssr. Simple export function, no async needed.

3. Create `src/types/database.ts` -- TypeScript types for the database tables:
   - `SiteSettings`: id (string), whatsapp (string), site_name (string), broker_name (string), updated_at (string)
   - `KeepAlive`: id (number), pinged_at (string)
   - Export a `Database` type following Supabase conventions (Tables with Row/Insert/Update types)

4. Create `src/middleware.ts` following the EXACT pattern from 01-RESEARCH.md Pattern 2:
   - Create Supabase client with request cookie handlers
   - Call `supabase.auth.getUser()` (NOT getSession -- security requirement)
   - If no user AND path starts with `/admin` -> redirect to `/login`
   - If user AND path is `/login` -> redirect to `/admin/imoveis`
   - Return `supabaseResponse` (never create a new NextResponse after Supabase client init)
   - Matcher config excluding static files

5. Create `supabase/schema.sql` with complete database schema:
   - `site_settings` table: id UUID PK default gen_random_uuid(), whatsapp TEXT NOT NULL DEFAULT '', site_name TEXT NOT NULL DEFAULT '', broker_name TEXT NOT NULL DEFAULT '', updated_at TIMESTAMPTZ DEFAULT now()
   - RLS enabled with policies: anon+authenticated SELECT, authenticated-only UPDATE and INSERT
   - Seed one empty settings row
   - `keep_alive` table: id BIGINT GENERATED BY DEFAULT AS IDENTITY PK, pinged_at TIMESTAMPTZ DEFAULT now()
   - RLS enabled with anon SELECT policy
   - `properties` table (schema ready for Phase 2): id UUID PK, title TEXT NOT NULL, description TEXT, price NUMERIC(12,2), property_type TEXT, bedrooms INT, bathrooms INT, area NUMERIC(10,2), address TEXT, neighborhood TEXT, city TEXT, state TEXT, latitude NUMERIC(10,7), longitude NUMERIC(10,7), status TEXT DEFAULT 'disponivel' CHECK (status IN ('disponivel','reservado','vendido')), featured BOOLEAN DEFAULT false, created_at TIMESTAMPTZ DEFAULT now(), updated_at TIMESTAMPTZ DEFAULT now()
   - Properties RLS: anon+authenticated SELECT, authenticated INSERT/UPDATE/DELETE
   - `property_images` table (schema ready for Phase 3): id UUID PK, property_id UUID REFERENCES properties ON DELETE CASCADE, storage_path TEXT NOT NULL, position INT DEFAULT 0, is_cover BOOLEAN DEFAULT false, created_at TIMESTAMPTZ DEFAULT now()
   - Property images RLS: same pattern

   NOTE: Properties and images tables are created now to avoid schema migrations later. Only site_settings and keep_alive are used in Phase 1.

6. Create `src/__tests__/middleware.test.ts`:
   - Test: unauthenticated request to /admin/imoveis redirects to /login
   - Test: authenticated request to /admin/imoveis passes through
   - Test: authenticated request to /login redirects to /admin/imoveis
   - Mock @supabase/ssr createServerClient and next/server NextResponse/NextRequest

7. Create `src/__tests__/validations.test.ts`:
   - Create `src/lib/validations/settings.ts` with Zod schema for settings form:
     - whatsapp: string, min 1 "WhatsApp e obrigatorio", regex for (XX) XXXXX-XXXX format
     - site_name: string, min 1 "Nome do site e obrigatorio"
     - broker_name: string, min 1 "Nome do corretor e obrigatorio"
   - Test: valid settings data passes validation
   - Test: empty whatsapp fails with "WhatsApp e obrigatorio"
   - Test: invalid phone format fails with "Formato invalido"
   - Test: empty site_name fails

IMPORTANT: Use `getUser()` never `getSession()` in any server-side code. All error messages in PT-BR.
